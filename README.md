# 🏙️ ObsidianCity3D — Backend

Converts an Obsidian vault into 3D-ready JSON data for a Three.js project.

## Quick Start

```bash
npm install

# Copy the environment file and update the path
cp .env.example .env
nano .env  # Set VAULT_PATH to your real vault

# Build once before the first run
npm run build
```

## Run

```bash
# Normal run
npm start

# Development mode with hot reload
npm run dev

# Tests
npm test
```

## Environment Requirements

- `VAULT_PATH` is required and must point to an existing Obsidian vault.
- `JWT_SECRET` is optional. If left empty, authentication stays disabled.
- `JWT_SECRET` must be at least 32 characters long if you want JWT auth enabled.

## API Endpoints

| Endpoint                  | Description                                |
| ------------------------- | ------------------------------------------ |
| `GET /api/health`         | Server health check                        |
| `GET /api/vault`          | Full vault data: cities, buildings, links  |
| `GET /api/vault/stats`    | Quick vault statistics                     |
| `GET /api/vault/note/:id` | Contents of a specific note                |

## `/api/vault` Response Format

```json
{
  "meta": {
    "vaultName": "MyVault",
    "totalCities": 3,
    "totalNotes": 42,
    "totalLinks": 87
  },
  "cities": [
    {
      "id": "المعلوماتية",
      "name": "المعلوماتية",
      "depth": 0,
      "position": { "x": -110, "z": 0, "y": 0 },
      "notes": [
        {
          "id": "react",
          "name": "React",
          "links": ["typescript", "node-js"],
          "linkCount": 2,
          "tags": ["frontend", "javascript"],
          "color": "#1a73e8",
          "height": 2
        }
      ],
      "subfolders": [ ... ]
    }
  ],
  "connections": [
    { "from": "react", "to": "typescript" },
    { "from": "react", "to": "node-js" }
  ]
}
```

## WebSocket Events

The server broadcasts events automatically when the vault changes:

```json
{ "type": "vault:change",      "event": "change", "noteId": "react" }
{ "type": "vault:newFolder",   "path": "/path/to/folder" }
{ "type": "vault:removeFolder","path": "/path/to/folder" }
```

## Conversion Rules

| Obsidian Item   | Three.js Meaning      |
| --------------- | --------------------- |
| Top-level folder| City                  |
| Subfolder       | District / village    |
| `.md` note      | Building              |
| `[[wikilinks]]` | Roads / connections   |
| Link count      | Building height       |
| `#tags`         | Building color / type |

## GitHub Publishing Notes

- Your local `.env` is not committed; use `.env.example` as the starting point.
- `coverage/` and `dist/` are generated by tests/builds and are excluded from Git.
